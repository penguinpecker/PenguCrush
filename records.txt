2026-05-18 — Research agents returned
  ▸ AGW session keys (CRITICAL): NO custom validator needed. Abstract
    provides one at 0x34ca1501FAE231cC2ebc995CE013Dbe882d7d081
    (same address on mainnet 2741 + testnet). Policy registry at
    0xfD20b9d7A406e2C4f5D6Df71ABE3Ee48B2EccC9F.
    API: agwClient.createSession({ session: SessionConfig }) for grant
    (one AGW prompt). createSessionClient or agwClient.toSessionClient
    for usage — drop-in replacement for regular client, no per-tx prompt.
    Policy fields: signer, expiresAt, feeLimit{Limit{type,limit,period}},
    callPolicies[]{target, selector, maxValuePerUse, valueLimit,
    constraints[]{index, condition, refValue, limit}}, transferPolicies[].
    Revocation: agwClient.revokeSessions({session}) — requires owner sig.
    Key gotchas: session key compromise drains full scope; one CallPolicy
    per (target, selector); period straddling possible — use Lifetime
    feeLimit for hard caps.
    → ADR-005 confirmed: 24h session, scoped to PenguCrushV2 non-payable
      selectors, feeLimit Lifetime 0.05 ETH, shop functions excluded so
      they trigger normal AGW prompt automatically.
    → Task #3 simplified: no PenguSessionKeyValidator contract needed.

  ▸ OpenZeppelin v5.6.1 production patterns:
    - Ownable2StepUpgradeable (not OwnableUpgradeable) for safer transfer
    - _disableInitializers() in constructor (mandatory)
    - __Ownable2Step_init() + __UUPSUpgradeable_init() + __EIP712_init()
      + __ReentrancyGuard_init() + __Pausable_init() chain in initialize
    - upgradeToAndCall(newImpl, "") — no upgradeTo() in v5
    - ERC-7201 namespaced storage preferred for new code OR flat storage
      + __gap[N] (pick one, don't mix)
    - PausableUpgradeable still standard; for partial pause use explicit
      bool shopPaused, gameplayPaused with custom modifiers
    - EIP-712: __EIP712_init("PenguCrush","1"), ECDSA.recover(digest,sig)
      single-arg form (rejects malleable high-s sigs)
    - SafeERC20.safeTransferFrom for USDC + hardcode decimals=6 with
      one-time assertion in initialize
    - Solidity 0.8.24: require(cond, CustomError()) NOT supported, must
      use if(!cond) revert CustomError(); pattern
    - Never call renounceOwnership on UUPS — locks impl forever

2. ARCHITECTURE DECISIONS
═══════════════════════════════════════════════════════════════════════════════

ADR-001 (2026-05-18) — Fresh deploy, not upgrade
  Decision: deploy PenguCrushV2 as a NEW proxy. Don't upgrade the existing
  v1 proxy at 0xAF2E…c517.
  Why: deployer for v1 is unrecoverable on this machine. v1 proxy has zero
  players anyway (silent-failure bug), so no data is lost.
  Trade-off: anyone with the v1 address hardcoded somewhere will break.
  Mitigation: legacy v1 proxy left on chain, frontend points only at v2.

ADR-002 (2026-05-18) — Items registry on chain, admin-mutable
  Decision: boosters, shards, currencies, lives all live in a single items
  registry (`mapping(bytes32 sku => ItemConfig)`), admin can add/disable
  without redeploying. Wheel config likewise stored on chain.
  Why: catalogue changes (new shard, new wheel reward) shouldn't need a
  contract upgrade. Upgrade reserved for actual code-shape changes.

ADR-003 (2026-05-18) — Lives + 8h regen authoritative on chain
  Decision: LifeAccount{regular, frozen, lastConsumedAt} in contract state.
  Regen computed virtually on read, materialized on write. Each tick fires
  LifeRegenerated event (via auto-claim from frontend on tick, plus hourly
  cron sweeper for offline users).
  Why: client-side regen is trivially editable in localStorage. Need on-
  chain truth for leaderboards, analytics, and any future monetization.

ADR-004 (2026-05-18) — Signed-quote oracle for prices
  Decision: shop SKUs are USD-denominated on chain. A backend price relayer
  (Supabase edge function) signs EIP-712 `Quote(sku, qty, ethWei,
  usdcMicros, validUntil, nonce)` payloads. Contract verifies signature +
  expiry before accepting payment.
  Why: no Chainlink/Pyth ETH/USD live on Abstract yet (to confirm). This
  pattern is chain-agnostic, oracle-free, and prevents stale-price griefing
  with a ~60s quote window.

ADR-005 (2026-05-18) — AGW session keys for in-game txs
  Decision: 24h session key, scoped to PenguCrushV2 address only, function
  selectors limited to non-payable game actions (submitLevel,
  consumeBooster, awardShards, claimRegen, etc.). Shop functions are
  excluded — they always require explicit AGW prompt because real $.
  Why: eliminates per-tx prompt UX, AGW-native primitive, owner can revoke.

#1 (open, 2026-05-18) — ALL chain writes silently failing
  Symptom: v1 proxy getPlayerCount() = 0 despite 5 real players in Supabase
    and 10 completed levels.
  Root cause hypothesis: src/onchain.js safeWrite() catches every error
    and returns null without surfacing to UI. Combined with possible AGW
    network/signing issues, the failure mode is invisible.
  Status: fixing via v2 deploy. New frontend will surface chain failures
    to a debug overlay (dev-mode) and a Supabase error_log table (prod).
  Notes: this is the failure mode flagged in user's memory feedback
    "Verify env vars + critical paths before shipping fail-closed."

    49 custom errors.

      Bytes.sol (EIP-5656, requires Cancun-aware solc 0.8.25+).

    ✗ zksolc warning that default codegen will become hard error.
      ✓ FIX: explicit `codegen: "yul"` in hardhat.config.cjs zksolc.settings.
    ✗ Used `bytes8` for "YYYY-MM-DD" day key — needs 10 chars, won't fit.
      ✓ FIX: replaced day/week keys with uint64 julian days throughout,
        deleted ~80 lines of date-formatting helpers + assembly bytes-cast.
    ✗ `if (msg.value != 0)` in non-payable USDC functions — solc rejects.
      ✓ FIX: removed; non-payable already auto-reverts on nonzero value.
    ✗ Legacy PenguCrush.sol + PenguCrushScores.sol had pre-existing
      duplicate-var-decl issues with 0.8.26.
